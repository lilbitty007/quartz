名称：- 企业微信内的 _吊车保险／信息费托管_ 智能客服
#### --构建一个基于知识库的智能合同生成工作流。通过“离线知识预处理”与“在线并行生成”相结合的架构，实现针对不同业务场景的、高质量的 合同即报价（保密协议）动态生成。




吊车这类大型工业机械的主流方案仍以“工程机械设备保险+CPE责任扩展”为核心，采用3‰基准费率加多重系数微调；但随着 IoT 与工业互联网的普及，按工况计费、融资租赁打包等创新模式正快速渗透，正在改变报价和风险管理的方式，也在无形中抬高了技术、数据与服务层面的行业准入门槛。

标注的文字数据集和部分保险产品介绍
基于RAG框架下 搭建平台内部客服知识管理系统--即吊车保险合同报价知识库设计 和 信息费托管服务的智能人工客服
基于dify工作流--保险方案这块主要是AI智能客服的自动回答（涵盖报价，合同合同生成）

- 文本：200 ~ 500 token 分块，条款号/险种作元数据标签，存向量库
    
- 模板化分块：对 PDF 使用 ragflow 的 _Paper_ 模板保证标题-正文完整性[6](https://deepseek.csdn.net/682446d1e47cbf761b6cfa96.html)
    

**Dify 工作流**

1. `Retriever` 节点：多路召回 + Re-ranking[3](https://aws.amazon.com/cn/blogs/china/production-grade-telesales-script-generation-tool-based-on-rag-and-dify/)
    
2. `LLM` 节点
    
    - 普通问答 → RAG 生成
        
    - 报价 → 函数调用 `getQuote(product, carInfo)` 计算费率
        
    - 合同 → 填充 docx 模板并返回下载链接
        
3. `Post-process`：引用标注，合规检查（防幻觉）
    
4. `Webhook`：结果回写企业微信消息
    

**效果指标**

- 首次解决率 ≥ 92 %（企业微信 SLA 建议值）[4](https://www.betteryeah.com/blog/enterprise-wechat-ai-customer-service-knowledge-base-guide)
    
- 响应 ≤ 3 s；RAG 片段数量 ≤ 6 条确保延迟<800 ms
